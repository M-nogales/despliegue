Easy Git Guide

Curso de Git para principiantes

Bienvenido al curso básico de Git. A continuación, se presentan los conceptos fundamentales:

1. Introducción

Git es un sistema de control de versiones distribuido que permite a los equipos de trabajo colaborar en el mismo proyecto sin conflictos.

2. Instalación

Descargar Git desde la página oficial y seguir los pasos del instalador. Sitio oficial de Git

3. Comandos básicos
  • git init - Inicializa un nuevo repositorio Git.
  • git add . - Agrega todos los cambios al área de preparación.
  • git commit -m "mensaje" - Guarda los cambios con un mensaje descriptivo.
  • git status - Muestra el estado actual del repositorio.
4. Trabajando con repositorios remotos
  • git remote add origin [url] - Conecta tu repositorio local con un repositorio remoto.
  • git push -u origin master - Sube tus cambios al repositorio remoto.
  • git pull - Trae cambios del repositorio remoto a tu repositorio local.
5. Consejos y mejores prácticas

Es vital hacer commits pequeños y descriptivos para que el historial del proyecto sea entendible. También es recomendable trabajar con ramas (branches) para organizar mejor las características y correcciones del proyecto.

Git Avanzado: Gestión de ramas

Las ramas en Git te permiten desarrollar características, arreglos y experimentos en paralelo sin afectar el flujo principal del proyecto.

1. Creación y cambio de ramas
  • git branch [nombre] - Crea una nueva rama.
  • git checkout [nombre] - Cambia a la rama especificada.
  • git checkout -b [nombre] - Crea y cambia a la nueva rama en un solo paso.
2. Fusionar ramas
  • git merge [nombre] - Fusiona la rama especificada con la rama actual.
3. Resolución de conflictos

Cuando dos ramas modifican el mismo fragmento de código y se intenta fusionar, ocurre un conflicto. Es necesario resolverlo manualmente antes de continuar.

4. Ramas remotas
  • git push origin [nombre] - Sube la rama al repositorio remoto.
  • git fetch - Trae ramas remotas que no existen localmente.
  • git pull - Actualiza la rama actual con la versión del repositorio remoto.
GitHub: Gestión de Repositorios

GitHub es una plataforma basada en la web que utiliza Git para el control de versiones. Permite la colaboración entre desarrolladores.

1. Creación de Repositorios

Desde el menú "New repository", puedes inicializar un nuevo repositorio, agregar un README, licencia y .gitignore.

2. Clonar Repositorios
  • git clone [URL del repositorio] - Clona el repositorio en tu máquina local.
3. Issues y Pull Requests

Los "Issues" permiten llevar un seguimiento de tareas, mejoras y bugs en tu proyecto. Los "Pull Requests" son propuestas de cambios que otros desarrolladores sugieren al proyecto original.

4. GitHub Actions y CI/CD

GitHub Actions permite automatizar flujos de trabajo. Puedes configurar acciones para pruebas automáticas, despliegues, y más.
